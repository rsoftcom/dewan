## Global Conventions

### Data types (PostgreSQL â†’ Prisma)

| PostgreSQL | Prisma | Usage |
|---|---|---|
| `UUID` | `String @id @default(uuid())` | All primary and foreign keys |
| `VARCHAR(n)` | `String @db.VarChar(n)` | Short bounded strings |
| `TEXT` | `String` | Unbounded strings, notes, reasons |
| `NUMERIC(12,2)` | `Decimal @db.Decimal(12,2)` | Prices, amounts, quantities |
| `BOOLEAN` | `Boolean` | Flags |
| `TIMESTAMPTZ` | `DateTime @db.Timestamptz` | All timestamps, always UTC |
| `DATE` | `DateTime @db.Date` | Calendar dates without time |
| `JSONB` | `Json` | Flexible metadata |
| Enum | `enum` (Prisma) | All status and type fields |

### Soft delete policy
Entities that support soft delete are marked with `status: active | inactive`. Physical deletion (`DELETE`) is **never used** on these entities. Hard deletes only apply to join tables (`product_category`, `recipe_item`) when the parent is removed.

### Tenant isolation
Every entity that belongs to a business carries `tenant_id`. The `TenantGuard` in NestJS injects `tenantId` from the JWT into every query via Prisma middleware, preventing cross-tenant data leaks at the application layer. Indexes on `tenant_id` are mandatory on every tenant-scoped table.
